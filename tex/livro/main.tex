\begin{dica}
\noindent\textbf{Dica ao professor \textemdash\ Introdução (Livro Didático):}

Um livro didático exige uma abordagem diferenciada. O objetivo aqui não é convencer uma banca, mas \textbf{engajar um aluno}.
\begin{itemize}
    \item Começe com exemplos do cotidiano (ex: como a Netflix usa dados, predições em esportes, detecção de fraude no cartão de crédito).
    \item Defina os termos-chave logo de cara e de forma simples.
    \item Apresente a ementa do livro: ``Neste capítulo você aprenderá A; no próximo, B.''

Sugere-se dividir este capítulo introdutório em 2 a 3 encontros de aula.

A \index{Ciência de Dados}Ciência de Dados é uma disciplina interdisciplinar que combina estatística, computação e conhecimento de domínio para extrair conhecimento de dados~\cite{provost2013}. Diferentemente da estatística clássica, a Ciência de Dados lida com volumes massivos de informação, frequentemente não estruturada, e emprega métodos computacionais escaláveis.

Conforme destacam \citeonline{hastie2009}, o objetivo central é transformar dados brutos em informação acionável. Este processo é frequentemente descrito pelo ciclo \index{CRISP-DM}CRISP-DM (\textit{Cross-Industry Standard Process for Data Mining}), ilustrado na Figura~\ref{fig:pipeline}.

\section{O que são Dados?}

Dados são registros de fenômenos observáveis. No contexto da Ciência de Dados, classificam-se em:

\begin{itemize}
    \item \textbf{Estruturados}: organizados em tabelas (linhas e colunas), como planilhas e bancos relacionais;
    \item \textbf{Semi-estruturados}: possuem alguma organização, como JSON e XML;
    \item \textbf{Não estruturados}: textos, imagens, áudio e vídeo.
\end{itemize}

\end{dica}

\index{Python}Python~\cite{mckinney2022} tornou-se a linguagem dominante em Ciência de Dados devido à sua sintaxe simples, ecossistema rico de bibliotecas e comunidade ativa. Este capítulo apresenta os fundamentos necessários para acompanhar o restante do livro.

A Equação~\ref{eq:media} define a média aritmética, uma das medidas mais utilizadas:

\begin{equation}
    \bar{x} = \frac{1}{n} \sum_{i=1}^{n} x_i
    \label{eq:media}
\end{equation}

\begin{dica}
\noindent\textbf{Dica ao aluno \textemdash\ A armadilha da média:}

Um erro muito comum em Ciência de Dados é achar que a \textbf{média} conta a história toda. A média é muito sensível a valores extremos (\textit{outliers}).
\begin{itemize}
    \item Imagine que em uma sala de aula de 10 pessoas, a renda média seja R\$ 2.000. Se Elon Musk entrar na sala, a renda média subirá para R\$ 1 bilhão. Isso representa os alunos? Não.
    \item Nestes casos (distribuições assimétricas), a \textbf{mediana} é muito mais confiável.
    \item Use \texttt{df.describe()} no \textit{pandas} para obter ambas simultaneamente.
\end{itemize}
\end{dica}

\begin{exercicio}
Carregue um dataset CSV de sua escolha usando \texttt{pandas.read\_csv()} e calcule a média, mediana e desvio padrão de cada coluna numérica. Compare os resultados e identifique possíveis \textit{outliers}.
\end{exercicio}

\begin{dica}
\noindent\textbf{Dica ao professor \textemdash\ Análise Exploratória de Dados (EDA):}

A EDA é a etapa mais criativa do pipeline e, paradoxalmente, a mais difícil de ensinar porque \textbf{não tem receita pronta}.
\begin{itemize}
    \item Em vez de pedir ``faça 5 gráficos'', peça aos alunos: ``formulem 3 perguntas sobre este dataset e tentem respondê-las usando gráficos''.
    \item Lembre-os do princípio de Tukey (1977): \textit{''Explorar é procurar sem saber exatamente o que se procura.''}
    \item Demonstre que gráficos simples (barras, dispersão) frequentemente revelam mais que visualizações complexas em 3D.
\end{itemize}
\end{dica}

A \index{Análise Exploratória}Análise Exploratória de Dados (EDA)~\cite{tukey1977} é o processo de investigar dados por meio de resumos estatísticos e representações gráficas. O objetivo é compreender a estrutura dos dados antes de aplicar modelos.

\section{Análise de Correlação}

A correlação mede a relação linear entre duas variáveis. O coeficiente de correlação de \index{correlação}Pearson ($r$) varia de $-1$ a $+1$, conforme a Equação~\ref{eq:pearson}:

\begin{equation}
    r_{xy} = \frac{\sum_{i=1}^{n} (x_i - \bar{x})(y_i - \bar{y})}{\sqrt{\sum_{i=1}^{n} (x_i - \bar{x})^2 \sum_{i=1}^{n} (y_i - \bar{y})^2}}
    \label{eq:pearson}
\end{equation}

\chapter{Pré-processamento de Dados}
% ============================================================

\begin{dica}
\noindent\textbf{Dica ao aluno \textemdash\ \textit{Garbage In, Garbage Out}:}

A dura realidade: o pré-processamento de dados vai consumir cerca de \textbf{80\% do seu tempo} em projetos reais.
\begin{itemize}
    \item Não existe modelo mágico que salve dados ruins. Se você colocar lixo na entrada, sairá lixo na predição (\textit{Garbage In, Garbage Out}).
    \item Dados ausentes têm significado? Muitas vezes um campo vazio (ex: não preencheu a renda) é uma informação mais útil para prever fraude do que a própria renda.
    \item Trate o pré-processamento como a fundação de um prédio. Se pular esta etapa, seu modelo vai desmoronar quando encontrar dados do mundo real.
\end{itemize}
\end{dica}

O \index{pré-processamento}pré-processamento transforma dados brutos em um formato adequado para modelagem. As principais etapas incluem tratamento de valores ausentes, \index{encoding}codificação de variáveis categóricas e normalização.

\section{Valores Ausentes}

Valores ausentes podem ser tratados por:
\begin{itemize}
    \item \textbf{Remoção}: eliminar linhas ou colunas com muitos valores faltantes;
    \item \textbf{Imputação}: preencher com média, mediana, moda ou valor previsto.
\end{itemize}

\section{Codificação de Variáveis Categóricas}

Algoritmos de \index{Machine Learning}ML exigem entrada numérica. Variáveis categóricas são convertidas via \textit{one-hot encoding} ou \textit{label encoding}.

\section{Normalização}

A normalização escala as variáveis para uma faixa comparável. Dois métodos comuns:

\begin{equation}
    \text{Min-Max: } x' = \frac{x - x_{\min}}{x_{\max} - x_{\min}}
    \label{eq:minmax}
\end{equation}

\begin{equation}
    \text{Z-Score: } x' = \frac{x - \bar{x}}{\sigma}
    \label{eq:zscore}
\end{equation}

\begin{dica}
\noindent\textbf{Dica ao aluno \textemdash\ A arte e a \textit{ciência} da avaliação:}

Avaliar um modelo é muito mais do que olhar para a Acurácia.
\begin{itemize}
    \item \textbf{O paradoxo da acurácia:} Se num banco só 1\% das transações for fraude, um modelo inútil que sempre diz ``NÃO É FRAUDE'' terá 99\% de acurácia.
    \item Para dados desbalanceados, você precisa entender a Matriz de Confusão, Precisão, Recall e a Curva ROC.
    \item \textbf{Um gráfico bem feito:} Um bom gráfico (como a Figura~\ref{fig:roc}) comunica em segundos o que uma tabela levaria páginas para explicar. Contudo, exija de si mesmo o básico: título, eixos nomeados, legenda e escala correta.
\end{itemize}
\end{dica}
