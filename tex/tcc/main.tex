\begin{resumo}
\noindent Este trabalho apresenta um estudo sobre a aplicação de técnicas de \textit{Machine Learning} para a previsão de \textit{churn} de clientes no setor de telecomunicações. Utilizando um pipeline reprodutível de Ciência de Dados, foram comparados três modelos preditivos: Regressão Logística, \textit{Random Forest} e \textit{LightGBM}. O modelo \textit{LightGBM} obteve o melhor desempenho, com AUC-ROC de 0,877. O repositório completo com dados, código e documentação está disponível para consulta e reprodução. Os resultados corroboram a literatura recente sobre a eficácia de métodos de \textit{ensemble} para problemas de classificação no domínio de telecomunicações.
\newline
\noindent \textbf{Palavras-chave:} \index{Ciência de Dados}Ciência de Dados. \textit{\index{Machine Learning}Machine Learning}. Previsão de \textit{Churn}. \index{Classificação}Classificação Binária. \index{Reprodutibilidade}Reprodutibilidade.
\end{resumo}

\begin{otherlanguage*}{english}
\begin{resumo}[Abstract]
\noindent This work presents a study on the application of Machine Learning techniques for customer churn prediction in the telecommunications sector. Using a reproducible Data Science pipeline, three predictive models were compared: Logistic Regression, Random Forest, and LightGBM. The LightGBM model achieved the best performance, with AUC-ROC of 0.877. 
\newline
\noindent \textbf{Keywords:} Data Science. Machine Learning. Churn Prediction. Binary Classification. Reproducibility.
\end{resumo}
\end{otherlanguage*}

No cenário atual de telecomunicações, a retenção de clientes tornou-se um diferencial competitivo crucial\index{retenção de clientes}. O fenômeno do \textit{churn}\footnote{A palavra \textit{churn} origina-se do inglês e significa ``agitar'' ou ``bater''. No contexto de negócios, metaforicamente descreve a ``agitação'' causada pela saída de clientes. O termo foi popularizado no setor de telecomunicações nos anos 1990 e hoje é utilizado em diversos segmentos como SaaS, bancos e seguros.} representa a taxa de cancelamento de serviços e impacta diretamente a receita das operadoras. Estima-se que adquirir um novo cliente custe de cinco a sete vezes mais do que reter um existente~\cite{provost2013}.

A \index{Ciência de Dados}Ciência de Dados emerge como ferramenta estratégica para identificar precocemente clientes com alto risco de cancelamento. Conforme destacado por \citeonline{wazlawick2024}, a formulação precisa do problema de pesquisa é o primeiro passo para o sucesso de um trabalho científico. Neste contexto, o presente trabalho busca responder: \textbf{é possível prever o \textit{churn} de clientes de telecomunicações utilizando modelos de \textit{Machine Learning} de forma reprodutível?}

\section{Aprendizado de Máquina Supervisionado}

O \index{Aprendizado Supervisionado}Aprendizado Supervisionado é um ramo da \index{Inteligência Artificial}Inteligência Artificial que se concentra no desenvolvimento de algoritmos capazes de aprender padrões a partir de dados rotulados~\cite{hastie2009}. Formalmente, dado um conjunto de treinamento $\mathcal{D} = \{(\mathbf{x}_i, y_i)\}_{i=1}^{n}$, onde $\mathbf{x}_i \in \mathbb{R}^p$ representa o vetor de características e $y_i \in \mathcal{Y}$ o rótulo associado, o objetivo é aprender uma função $f: \mathbb{R}^p \to \mathcal{Y}$ que generalize bem para novos exemplos.

O erro de generalização pode ser decomposto em três componentes fundamentais, conforme a Equação~\ref{eq:bias_variance}:

\begin{equation}
    \mathbb{E}[(y - \hat{f}(\mathbf{x}))^2] = \underbrace{\text{Bias}[\hat{f}(\mathbf{x})]^2}_{\text{viés}} + \underbrace{\text{Var}[\hat{f}(\mathbf{x})]}_{\text{variância}} + \underbrace{\sigma^2}_{\text{ruído}}
    \label{eq:bias_variance}
\end{equation}

Esta decomposição, conhecida como \textit{trade-off} viés-variância~\cite{hastie2009}, é central para a compreensão do comportamento dos algoritmos de aprendizado.

\section{Regressão Logística}

\index{Regressão Logística}Regressão Logística é um modelo linear generalizado que estima a probabilidade de pertencimento a uma classe~\cite{geron2022}. Sua formulação matemática é dada pela Equação~\ref{eq:logistic}:

\begin{equation}
    P(Y = 1 \mid \mathbf{X}) = \frac{1}{1 + e^{-(\beta_0 + \boldsymbol{\beta}^T \mathbf{X})}}
    \label{eq:logistic}
\end{equation}

\section{Random Forest}

O \index{Random Forest}\textit{Random Forest}~\citeonline{breiman2001} é um algoritmo de \textit{ensemble} que combina múltiplas árvores de decisão para melhorar a acurácia e controlar o \textit{overfitting}. A importância das variáveis pode ser visualizada na Figura~\ref{fig:importancia}.

\section{LightGBM}

\index{LightGBM}\textit{LightGBM}~\cite{ke2017lightgbm} é uma implementação eficiente de \textit{Gradient Boosting} que introduz duas inovações principais: \textit{Gradient-based One-Side Sampling} (GOSS) e \textit{Exclusive Feature Bundling} (EFB). Estas otimizações permitem treinamento mais rápido sem sacrificar a acurácia~\cite{geron2022}.

As métricas de avaliação para classificação, como AUC-ROC e \textit{F1-Score}, são discutidas em detalhes por \citeonline{kuhn2013}.

\section{Pré-processamento}

As etapas de pré-processamento incluíram: imputação de valores ausentes, tratamento de \textit{outliers} por winsorização, criação de \textit{features} derivadas (renda por produto, taxa de reclamação por tempo de conta), codificação \textit{one-hot} e normalização com \textit{StandardScaler}.

\section{Treinamento e Validação}

Foram treinados três modelos: Regressão Logística ($C=1.0$), \textit{Random Forest} (200 árvores) e \textit{LightGBM} (200 iterações). A validação utilizou \textit{Stratified K-Fold} ($K=5$) com divisão 80\%/20\%.
