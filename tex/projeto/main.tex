\section{Introdução}

\begin{dica}
\noindent\textbf{Dica ao autor \textemdash\ Introdução de Projeto de Pesquisa:}

Um projeto de pesquisa não apresenta resultados, ele \textbf{vende uma ideia}. A introdução deve convencer o avaliador de que o tema é importante e merece ser estudado.
\begin{itemize}
    \item \textbf{Cenário:} Qual é o contexto atual do tema? (Use dados reais para ancorar a discussão).
    \item \textbf{O Problema:} O que está quebrado, incompleto ou ineficiente neste cenário?
    \item \textbf{A Proposta:} O que você vai fazer para resolver ou mitigar este problema?
\end{itemize}
Seja persuasivo, mas cientificamente preciso.
\end{dica}

A retenção de clientes é um dos principais desafios no setor de telecomunicações. O fenômeno do \textit{churn} representa perda significativa de receita, sendo que adquirir um novo cliente custa de cinco a sete vezes mais do que reter um existente \cite{provost2013}. Nesse contexto, a aplicação de técnicas de Aprendizado de Máquina (\textit{Machine Learning}) tem se mostrado eficaz na identificação precoce de clientes com alto risco de evasão \cite{hastie2009}.

Este projeto de pesquisa propõe o desenvolvimento de um sistema preditivo de \textit{churn} baseado em modelos supervisionados de classificação, seguindo um pipeline reprodutível de Ciência de Dados.

\section{Problemática}

\begin{dica}
\noindent\textbf{Dica ao autor \textemdash\ A Pergunta de Pesquisa:}

Todo projeto científico nasce de uma pergunta.
\begin{itemize}
    \item Formule o problema como uma \textbf{pergunta explícita} (terminando com pondo de interrogação).
    \item A pergunta deve ser clara, precisa e empírica (possível de ser respondida com dados, não com opiniões).
    \item Exemplo ruim: "Qual o melhor modelo de IA?"
    \item Exemplo bom: "Qual modelo supervisionado (LogReg, RF, LGBM) apresenta maior AUC-ROC na previsão de churn em operadoras de telefonia móvel?"
\end{itemize}
\end{dica}

As empresas de telecomunicações enfrentam dificuldades em identificar proativamente clientes propensos ao cancelamento. A ausência de modelos preditivos eficazes leva a ações reativas e custosas. Diante disso, questiona-se: \textbf{como utilizar dados históricos de clientes para construir modelos preditivos capazes de antecipar o \textit{churn} com alta acurácia?}

\section{Justificativa}

\begin{dica}
\noindent\textbf{Dica ao autor \textemdash\ O "Por quê?":}

A justificativa responde à pergunta: \textbf{Por que este estudo deve ser realizado?}
\begin{itemize}
    \item \textbf{Relevância Acadêmica:} Qual lacuna na literatura este trabalho vai preencher?
    \item \textbf{Relevância Social/Econômica:} Quem se beneficia com os resultados? (ex: empresas reduzem custos, pacientes recebem diagnóstico mais rápido).
    \item \textbf{Viabilidade:} Mostre brevemente que o projeto é exequível no tempo proposto.
\end{itemize}
\end{dica}

A relevância deste projeto reside na oportunidade de reduzir custos operacionais por meio da retenção preventiva de clientes. Além disso, a construção de um pipeline reprodutível contribui para a literatura ao fornecer uma referência metodológica para trabalhos futuros. Do ponto de vista acadêmico, o projeto integra conceitos de Ciência de Dados, Estatística e Aprendizado de Máquina, consolidando a formação do pesquisador.

\section{Objetivos}

\subsection{Objetivo Geral}

Desenvolver e comparar modelos de \textit{Machine Learning} para a previsão de \textit{churn} de clientes em telecomunicações, com ênfase na reprodutibilidade dos resultados.

\subsection{Objetivos Específicos}

\begin{enumerate}
    \item Realizar análise exploratória dos dados de clientes para identificar padrões e \textit{outliers};
    \item Aplicar técnicas de pré-processamento e engenharia de \textit{features};
    \item Treinar e validar modelos de Regressão Logística, \textit{Random Forest} e \textit{LightGBM};
    \item Comparar o desempenho dos modelos utilizando métricas como AUC-ROC, F1-Score e acurácia;
    \item Documentar o pipeline de forma que os experimentos sejam integralmente reprodutíveis.
\end{enumerate}

\section{Revisão de Literatura}

\begin{dica}
\noindent\textbf{Dica ao autor \textemdash\ Estado da Arte no Projeto:}

No projeto, a revisão de literatura não precisa ser exaustiva como na tese final, mas deve ser \textbf{estratégica}.
\begin{itemize}
    \item Demonstre que você conhece os principais autores do campo.
    \item Foque em mostrar o que \textbf{já foi feito} e, principalmente, o que \textbf{ainda não foi feito} (a lacuna que seu projeto vai preencher).
    \item Se você propõe usar LightGBM para churn, cite quem já usou Random Forest para o mesmo fim e quem já usou LightGBM para outros fins. Você está conectando os pontos.
\end{itemize}
\end{dica}

A previsão de \textit{churn} tem sido amplamente estudada na literatura. \citeonline{ahmad2019} aplicaram modelos de \textit{Machine Learning} em uma plataforma de \textit{Big Data} para prever evasão em telecomunicações, obtendo resultados promissores com \textit{Random Forest} e \textit{Gradient Boosting}.

No contexto brasileiro, \citeonline{silva2022} desenvolveram um modelo preditivo para inadimplência bancária, demonstrando a aplicabilidade das técnicas em dados nacionais. Estudos recentes~\cite{pecanha2024} destacam a importância da reprodutibilidade em experimentos de Ciência de Dados, propondo \textit{frameworks} para versionamento de dados e modelos.

Os fundamentos teóricos dos algoritmos utilizados --- Regressão Logística, \textit{Random Forest} e \textit{LightGBM} --- são amplamente discutidos por \citeonline{hastie2009}, \citeonline{breiman2001} e \citeonline{ke2017lightgbm}, respectivamente.

\subsection{Pré-processamento}

As etapas de pré-processamento incluirão: imputação de valores ausentes, tratamento de \textit{outliers}, codificação de variáveis categóricas e normalização dos dados.

\subsection{Treinamento e Validação}

Serão treinados três modelos --- Regressão Logística, \textit{Random Forest} e \textit{LightGBM} --- com validação cruzada \textit{Stratified K-Fold} ($K = 5$) e divisão treino/teste de 80\%/20\%.

\subsection{Avaliação}

Os modelos serão comparados pelas métricas de AUC-ROC, F1-Score e acurácia, com análise da matriz de confusão do melhor modelo.
